# Agent Harness for Reflective Piano Analysis

This module implements a lightweight multi-agent pipeline for comparing an expert piano recording with a learner recording without converting audio into a score.

## Design

The harness runs a shared signal-analysis stage, then multiple agents inspect the result from different perspectives:

- `SignalOverviewAgent`: global duration, tempo proxy, loudness, alignment quality.
- `RubatoTimingAgent`: local tempo elasticity and timing profile.
- `DynamicsAgent`: loudness contour and dynamic contrast.
- `StructureAlignmentAgent`: aligned segment similarity and moments worth relistening.
- `ReflectionPromptAgent`: pedagogical prompts from measured differences.
- `OpenAISynthesisAgent`: uses OpenAI SDK to synthesize a Chinese reflective critique from all agent evidence. If `OPENAI_API_KEY` is missing or the API fails, it falls back to a local template report.

## LLM Config

The harness reads LLM settings from a local JSON config. Default lookup order:

1. `scripts/agent_harness/configs/llm.local.json`
2. `scripts/agent_harness/configs/llm.example.json`

Use `scripts/agent_harness/configs/llm.example.json` as the template. Do not put real API keys in the example file.

Minimal local config:

```json
{
  "default_profile": "openai_default",
  "profiles": {
    "openai_default": {
      "provider": "openai",
      "base_url": "https://api.openai.com/v1",
      "api_key_env": "OPENAI_API_KEY",
      "api_key": "",
      "models": {
        "synthesis": "gpt-4o-mini"
      }
    }
  }
}
```

Rules:

- `api_key` wins if non-empty.
- Otherwise the harness reads the environment variable named by `api_key_env`.
- `base_url` supports OpenAI-compatible gateways and local OpenAI-compatible inference services.
- `models.synthesis` is the default model for the final feedback agent.
- CLI `--model` and Gradio "模型 ID 覆盖" override `models.synthesis`.
- Missing config or missing API key does not crash the system; it falls back to the local template report.

## Boundary

The system does not perform audio-to-score transcription. It compares audio performances as performances, then asks the LLM to reason only from measured audio evidence and explicitly stated uncertainty.
